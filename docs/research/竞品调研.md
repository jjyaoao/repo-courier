| **微信公众号**                 | 本次官方材料中未发现明确、成熟的一等微信公众号 Connector                                                           | 无明确支持者；BettaFish 虽覆盖大量中文社交平台，但公开 README 主要列举微博、小红书、TikTok、快手等                      | 搜狗微信搜索、公众号 RSS 转换、授权抓取或用户配置公众号列表  | **这是当前竞品最明显的空白之一。**可将“指定公众号白名单＋文章历史索引＋增量监控”作为核心卖点；需重点评估授权、反爬和内容合规                                   |
| **中文及国际社交平台**         | 微博、小红书、TikTok、快手及大量用户评论、短视频内容                                                               | BettaFish                                                                                                               | 专用爬虫、平台搜索、多模态解析                               | 适用于舆情和趋势监测；技术情报产品可选择性支持知乎、掘金、CSDN、Bilibili、X 等来源                                                                               |
| **新闻、博客和出版物**         | 超过一万个新闻源、博客和出版站点；实时网络索引                                                                     | Newsletter AI、Perplexity、GPT Researcher、Open Deep Research、Kimi                                                     | Tavily、Perplexity、Google 等搜索索引                        | 需要加入时效性、媒体可信度、内容重复度和转载链识别，避免大量低质量转载污染报告                                                                                   |
| **代码仓库**                   | GitHub 仓库中的代码、README 和文档；本地代码目录；其他代码托管平台可经 MCP 扩展                                    | OpenAI Deep Research、GPT Researcher；Gemini、Open Deep Research、DeerFlow 可通过 MCP 接入                              | GitHub Connector、GitHub MCP、GitHub API、仓库克隆与本地解析 | 多数竞品只是“读取 GitHub”，并未专门提供“项目发现＋Star/提交活跃度＋Issue/Release＋技术栈分析”的完整能力；这是你的另一项核心差异化 ([OpenAI Help Center][18]) |
| **企业文档与私有数据**         | Google Drive、SharePoint、Gmail、Google Calendar、Google Chat、FactSet、PitchBook、Scholar Gateway、企业内部数据库 | OpenAI、Gemini、Claude、BettaFish                                                                                       | OAuth Connector、MCP、企业 API、数据库 Adapter               | 应采用用户权限继承和只读访问；私域内容需单独记录访问范围、来源身份和引用权限                                                                                     |
| **用户上传文件**               | PDF、Word、Excel、CSV、Markdown、PPT、TXT、图片、Google Sheets、Drive URL                                          | OpenAI、Gemini、NotebookLM、GPT Researcher、Tongyi、DeerFlow                                                            | 上传、对象存储、文件解析、向量索引                           | 应统一文件解析层，并保留页码、工作表、段落和幻灯片等精确定位信息，方便报告引用                                                                                   |
| **用户历史与偏好数据**         | 阅读记录、评分、主题兴趣、参与度、历史 Newsletter                                                                  | Newsletter AI                                                                                                           | Redis、向量数据库、用户级 RAG                                | 可用于定时报告个性化，但应与公共研究语料隔离，提供清除和关闭个性化的能力                                                                                         |
| **自定义工具与任意业务数据源** | Remote MCP、Python Function、数据库、企业 API、自定义 Skill                                                        | Gemini、GPT Researcher、Open Deep Research、DeerFlow、Claude Connectors                                                 | MCP、Skill、Function Calling                                 | 推荐将 MCP 作为统一扩展协议，同时保留原生 Adapter，以便对关键数据源实施更精细的缓存、限流和去重                                                                  |

### 3.2 对目标项目最重要的数据源结论

| 结论                                  | 调研发现                                                                              | 建议                                                                                                          |
| ------------------------------------- | ------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------- |
| 学术论文接入已有成熟参考              | Paper Search MCP 已覆盖二十多个来源，并实现 DOI、去重和全文回退                       | 首版可直接参考其 Connector 和统一数据模型，不必重新从零封装全部论文平台                                       |
| 官方博客仍缺少专门化管理              | 主流 Deep Research 产品主要依赖综合搜索或域名限制                                     | 建立 OpenAI、Anthropic、Google DeepMind、Kimi、阿里、字节等官方站点注册表，并结合 RSS、Sitemap 和更新时间监测 |
| 微信公众号是明显市场空白              | 被调研项目没有公开展示成熟的一等公众号检索连接器                                      | 以“用户配置公众号列表＋增量文章索引＋来源可信度标签”为特色，但必须处理授权与抓取合规                        |
| GitHub 多为通用读取，而非项目情报分析 | 现有项目常通过 GitHub Connector 或 MCP 获取仓库内容                                   | 增加仓库发现、Star 趋势、Release、Commit、Issue、贡献者、License、语言和依赖分析                              |
| 定时任务并非 Deep Research 标配       | DeerFlow、Newsletter AI 明确提供调度；Gemini Deep Research Max 适合 Cron 异步工作流   | 将调度作为独立基础模块，支持一次、Cron、条件触发、失败重试、去重和变更摘要                                    |
| HTML 报告是可形成优势的功能           | Kimi 和 Mind2Report 已提供动态或静态 HTML；多数产品仍以聊天报告、Markdown 或 PDF 为主 | 报告应支持章节导航、图表、来源卡片、时间线、代码仓库卡片、论文卡片、差异高亮和公开/私有分享                   |
| 多源融合不能只是结果拼接              | 商业产品和研究系统都强调计划、记忆、反思与来源验证                                    | 增加来源标准化、实体去重、交叉验证、矛盾检测、来源可信度和时间有效性评估                                      |
